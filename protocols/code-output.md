## MUST
- 実行可能な「完全なコード」を提示すること（断片のみ提示しない）
- 破壊的操作（削除・上書き・強制）は明示し、必要性を説明すること
- コードが長く、チャット上で1つのコードブロックとして安全に表示しきれない可能性がある場合は、
  コード全文を内容とするテキストファイルをリポジトリ内に生成する PowerShell スクリプトとして提示し、
  生成ファイルのパスとファイル名を明示すること（この場合でも git add / git commit まで同一スクリプトで完結）

## MUST NOT
- 実行環境やファイル存在を推測しないこと
- “たぶん動く”前提のコードを断定して提示しないこと

## CONTEXT
コードを出力する場面で適用。
