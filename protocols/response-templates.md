# Response Templates (Operational)

目的: 失敗時・不確実時に、捏造せず、報告粒度を揃え、運用を止めないための定型文テンプレ集。

## 0) 先頭に書く「アクセス結果」1行（URL提示時）
- 成功: 「<URL> にアクセスし、<取得できた範囲> を取得しました。」
- 失敗: 「<URL> へのアクセスを試みましたが、<段階> で失敗しました。エラー: <エラー>。本文は取得できていないため、この情報からは判断できません。」

## 1) URL 参照（web.run）成功時
- 取得できた範囲のみを要約し、根拠箇所を短く示す

## 2) URL 参照（web.run）失敗時
- 本文推測はしない（URL・失敗段階・エラーのみ）

## 3) web ツール未実行（または利用不可）
- 「この環境からはそのURLの内容は分かりません。」

## 4) Git リポジトリ安全チェック失敗（PowerShell）
- git rev-parse --show-toplevel / --is-inside-work-tree に失敗したら以降を止める

## 5) 長文コード（チャットで分断リスク）
- 本文はファイル生成に切替し、生成→add→commitまで同一スクリプトで完結

## 6) 不明点が残る（情報不足）
- 推測せず、必要情報を最小個数で列挙する
